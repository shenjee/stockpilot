# T+0 Chart Spike

验证 T+0 助手图表引擎能力的技术 Spike。

## 架构目标

- ✅ 逻辑时间轴：非交易时段（午休、隔夜、周末、停牌）不显示空槽
- ✅ 5 分钟图表组：价格图 + VOL + MACD，共享同一时间轴
- ✅ 跟随最新 / 手工浏览双状态
- ✅ 宽度驱动的满轴显示（容器宽度 -> 显示 N 根 K 线）
- ✅ 布局切换保持视口状态
- ✅ 回放截断（不显示 T 之后的数据）
- ✅ BOLL 指标
- ✅ 增量更新不重置手动浏览位置

## 技术选型

当前首选：**Lightweight Charts**

验证目标：
- 多图表同步（十字光标、可见范围）
- 性能（500 根 K 线流畅交互）
- 包体大小
- 自定义 overlay 能力（中枢、买卖点等）

## 验证清单

### 1. 数据确定性
- [ ] 500 根 K 线每次生成相同
- [ ] 跨交易日连续排列
- [ ] 非交易时段没有空槽

### 2. 状态转换
- [ ] 初始状态：跟随最新
- [ ] 手动拖动后：切换到手工浏览
- [ ] 拖动回最新边缘：恢复跟随
- [ ] 增量更新时：手工浏览位置不变
- [ ] 跟随模式下：增量更新自动滚动

### 3. 三图同步
- [ ] 价格图拖动时 VOL、MACD 同步
- [ ] VOL 拖动时价格图、MACD 同步
- [ ] 十字光标在三图中对齐
- [ ] 缩放操作保持同步

### 4. 宽度驱动满轴
- [ ] 容器变宽时：显示更多 K 线（跟随模式）
- [ ] 容器变窄时：显示更少 K 线（跟随模式）
- [ ] 手工浏览模式下：视口范围不随容器宽度变化

### 5. 回放截断
- [ ] 截断后不显示 T 时间之后的数据
- [ ] 视口自动调整到截断后的最新数据
- [ ] 截断后的指标计算正确

### 6. 性能指标
- [ ] 初始渲染时间 < 100ms
- [ ] 拖动/缩放帧率 > 30fps
- [ ] 增量更新延迟 < 50ms
- [ ] 包体大小 < 50KB gzipped

## 验收标准

所有验证项通过后，即可将图表模块设计正式纳入 T+0 助手的架构基线。
