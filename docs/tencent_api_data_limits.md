# 腾讯财经API数据范围限制

## 验证时间
2026年7月13日

## 各时间级别能获取的历史数据范围

| 级别 | 最早可获取日期 | 大约可获取历史 |
|------|--------------|--------------|
| **1分钟** | 约最近3个月 | ~90天 |
| **5分钟** | **2026-01-13** | **约6个月** |
| **15分钟** | 约最近7个月 | ~210天 |
| **30分钟** | **2025-11-17** | **约8个月** |
| **60分钟** | 约最近10个月 | ~300天 |
| **日线** | 约2024年6月 | **约2年** |
| **周线/月线** | 通常更久 | 多年 |

## 技术说明

腾讯财经分钟K线API采用分页方式获取，每次最多返回800根。即使继续向后翻页，超过上述时间范围的数据已被腾讯服务器删除，无法获取。

这个限制会直接影响：
1. **缠论多级别递归** - 低级别的5分钟/30分钟数据可能比期望的时间范围短
2. **级别对齐** - 不同时间级别的数据起始时间可能不一致，系统已自动对齐到交集范围
3. **长周期分析** - 如果需要分析一年以上的分钟级别走势，需要考虑其他数据源

## 数据源替代方案

如果需要更长历史的分钟数据，可以考虑：
- 购买商业数据服务商（如Tushare Pro、JoinQuant、RiceQuant等）
- 使用其他免费数据源（但数据质量可能参差不齐）
- 自建数据采集系统持续积累数据
